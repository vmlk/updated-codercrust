Binary Tree Data Structure -

Last Updated: 7july 2021


A Binary Tree is a hierarchical data structure in which each node has atmost two children generally reffered as left child and right child. User will create node by user defined data type.


Each node of binary tree contains three components:
    1. Pointer to right subtree
    2. Pointer to left subtree
    3. Data element

Binary Tree digrammatic representation -



Top most node of the tree is known as root node and if the root node is empty then the root node is pointed by NULL pointer




Types of Binary Tree based on structure-

    1. Rooted binary tree: It has a root node and every node has atmost two children.

    2. Full Binary tree: It is a tree in which every node in the tree has either 0 or 2 children.
        a. The number of nodes, n, in a full binary tree is atleast n = 2h – 1, and atmost n = 2h+1 – 1, where h is the height of the tree.
        b. The number of leaf nodes l, in a full binary tree is number, L of internal nodes + 1, i.e, l = L+1.

    3.Complete binary tree: It is a binary tree in which every level, except possibly the last, is completely filled, and all nodes are as far left as possible.
        a. The number of internal nodes in a complete binary tree of n nodes is floor(n/2).


    4. Balanced binary tree: A binary tree is height balanced if it satisfies the following constraints:
        a. The left and right subtrees' heights differ by at most one, AND
        b. The left subtree is balanced, AND
        c. The right subtree is balanced
        d. An empty tree is height balanced.
        e. The height of a balanced binary tree is O(Log n) where n is number of nodes.


    5. Degenarate tree: It is a tree is where each parent node has only one child node. It behaves like a linked list.
